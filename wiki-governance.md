# Wiki Governance

## Why Other Repository Wikis Are Disabled

The STO Info project uses a centralised Wiki repository (`sto-info-wiki`) as a navigation and orientation layer.

Wikis are disabled in other repositories to prevent:

- Documentation fragmentation
- Content duplication
- Version drift
- Confusion about canonical sources

---

## Documentation Model

STO Info follows a clear separation:

- Repository `/docs` folders  
  → Canonical, versioned technical documentation  
  → Protected by branch rules  
  → Reviewed via pull requests  

- Central Wiki  
  → High-level overview  
  → Cross-repository navigation  
  → Roadmap and concept summaries  
  → Onboarding context  

---

## Governance Principles

This structure ensures:

- Documentation is reviewed alongside code
- Security posture remains intact
- No bypass of branch protection rules
- Clear audit trail for technical changes
- Centralised discoverability

---

## Change Policy

Changes to the central Wiki should:

- Remain high-level
- Avoid implementation detail
- Link to repository documentation
- Respect intellectual property boundaries
- Avoid operationally sensitive information

---

This structure supports clarity, maintainability, and long-term governance of the STO Info ecosystem.
